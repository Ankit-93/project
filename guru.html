Building an AI-Powered Knowledge Repository and Search Engine for Enterprise Code and Documents

AI-Powered Knowledge Repository and Document Classification

Project Overview

This project aims to build a centralized knowledge repository for enterprise documents, enabling efficient search and retrieval using AI-powered semantic understanding and classification. By leveraging deep learning models such as Conv1D and LSTM for document classification, the system enhances document accessibility, organization, and retrieval accuracy.

Key Components and Functionality

Data Ingestion and Processing

  • Crawler & Delta: Collects documents from various sources like SharePoint and S3 buckets, focusing on incremental updates.
  • Nightly Document Delta Job Scheduler: Automates the crawling and updating process.
  • Document Processing Pipeline: Handles document format conversion, metadata extraction, and text preprocessing.
  • Data Extraction/OCR: Utilizes Textract and openpyxl for extracting text, including OCR for scanned documents.
  • Snippet Creation: Breaks down documents into smaller, searchable chunks for efficient retrieval.

AI-Driven Document Classification

Model Choice: Conv1D + LSTM

To improve document categorization and retrieval, we implemented a hybrid deep learning model combining 1D Convolutional Neural Networks (Conv1D) and Long Short-Term Memory (LSTM) networks.

Working Mechanism

  1. Text Preprocessing:
    • Tokenization using TensorFlow/Keras Tokenizer.
    • Padding to ensure uniform input length.
    • Word embedding with pre-trained word vectors (e.g., GloVe).
  2. Feature Extraction using Conv1D:
    • Convolutional layers capture local patterns in text.
    • Max-pooling reduces dimensionality and retains essential features.
  3. Contextual Learning using LSTM:
    • LSTM layers capture long-term dependencies in document content.
    • Outputs sequential context-aware representations.
  4. Classification Layer:
    • Fully connected (Dense) layer with Softmax activation for multi-class document classification.
    • Optimized using Categorical Cross-Entropy Loss and Adam optimizer.

Challenges & Solutions

  • Handling Diverse Document Types: Used custom embeddings and pre-trained word vectors for better text representation.
  • Class Imbalance: Applied class weighting and data augmentation (synonym replacement, paraphrasing) to balance dataset.
  • Scalability for Large Volumes: Optimized model batch processing and integrated with cloud-based storage (S3, MongoDB) for large-scale processing.

Impact and Results

  • Improved Classification Accuracy: Achieved ~92% accuracy in document classification across multiple categories.
  • Enhanced Search Efficiency: Semantic classification enabled more relevant and precise search results.
  • Scalable Architecture: The system is designed to handle large volumes of enterprise documents, ensuring smooth integration with existing workflows.

Conclusion

This project successfully implemented an AI-powered document classification system using Conv1D and LSTM, significantly improving the organization and retrieval of enterprise documents. Future work includes enhancing semantic search with transformer-based models (e.g., BERT, RoBERTa) for even more accurate classification and retrieval.
